Visão Geral de Funcionamento

A biblioteca segue um fluxo de execução simples, mas poderoso, para transformar elementos HTML “declarativos” em componentes totalmente funcionais no navegador.

Identificação dos componentes no DOM

No carregamento da página, a função initialize() varre o documento em busca de elementos que possuam uma classe que comece com component-.

Exemplo

          
            div 
            class='component-tab' 
            data-contents='Conteúdo 1:Conteúdo 2' 
            data-btn-names='Aba 1,Aba 2'
            div

Neste caso, o elemento será identificado como um componente de abas (component-tab)

.

Definição do componente

A função define(element) recebe o elemento encontrado e verifica qual modelo de componente corresponde a ele, usando findByName().

Esse modelo contém informações como:

  • Tipo de elemento HTML (div, img, table, etc.).
  • Classes CSS padrão.
  • Propriedades específicas (ex.: src para imagens, placeholder para inputs).
  • Estrutura de filhos (children).

Construção dinâmica

A função buildComponents() cria o elemento final baseado nas propriedades do modelo e nos atributos data-* definidos no HTML original.

Durante essa etapa, o componente:

  • Aplica classes CSS.
  • Define atributos (src, id, type, etc.).
  • Gera elementos filhos automaticamente.
  • Adiciona conteúdos dinâmicos (ex.: linhas de tabela, slides, links, abas).

Substituição do elemento original

Após ser totalmente montado, o novo componente substitui o elemento original no DOM.

Isso garante que o HTML final exibido ao usuário seja otimizado e funcional, sem que o desenvolvedor precise escrever toda a estrutura manualmente.

Renderização de tipos especiais

A biblioteca possui lógica personalizada para alguns componentes:

  • component-table → Gera e automaticamente a partir de data-th e data-td.
  • component-slide → Monta slides de imagens com a primeira imagem ativa.
  • component-breadcrumb → Cria navegação hierárquica (links em lista ordenada).
  • component-tab → Cria um sistema de abas com botões e conteúdos correspondentes.

Resumo visual do fluxo

  • Encontrar → Procurar no DOM classes que começam com component-.
  • Associar → Localizar o modelo de configuração do componente.
  • Construir → Criar elemento com base nos atributos e propriedades.
  • Substituir → Trocar o HTML original pelo componente montado.
  • Renderizar → Aplicar comportamentos e estilos interativos.
